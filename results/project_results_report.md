# 京东用户行为分析项目结果报告

## 1. 数据处理

完成原始用户行为数据的读取、清洗、重复值处理、缺失值检查、时间格式统一和数据类型优化，并将处理后的数据保存为 Parquet 文件。

## 2. 特征工程

完成用户特征、商品特征、时间特征、行为序列特征、业务特征和SVD隐式特征构建，并合并生成建模特征表。

## 3. 样本构建

以用户未来7天是否发生购买行为作为目标变量：

- `label = 1`：未来7天发生购买；
- `label = 0`：未来7天未发生购买。

当前特征表为用户级特征表，因此目标变量暂按用户级构建。

## 4. 数据集划分

采用分层抽样，按以下比例划分数据集：

- 训练集：70%
- 验证集：20%
- 测试集：10%

分层抽样保证三个数据集中的正负样本比例基本一致。

## 5. 类别不平衡处理

完成以下三种方案：

- 欠采样；
- SMOTE过采样；
- 类别权重调整。

欠采样通过减少未购买样本实现平衡；SMOTE根据购买样本生成合成样本；类别权重不改变数据量，而是在模型训练时提高购买样本的重要性。

## 6. 特征筛选

通过目标变量相关性和随机森林特征重要性分析，筛选主要特征，并保存最终建模数据集。

## 7. 当前输出

主要结果包括：

- 清洗后的用户行为数据；
- 用户、商品、时间和业务特征表；
- 最终建模数据集；
- 训练集、验证集和测试集；
- SMOTE训练数据集；
- 特征重要性结果；
- 特征与标签相关性结果。

## 8. 下一阶段

下一阶段将进行逻辑回归、XGBoost和LightGBM模型训练，并对比AUC、准确率、召回率和F1分数。



# 传统机器学习模型构建与调优报告

## 1. 项目目标

本阶段基于清洗和特征工程后的京东用户行为数据，构建用户未来7天是否购买的二分类预测模型。

按照任务要求，计划实现以下三种传统机器学习模型：

1. Logistic Regression
2. XGBoost
3. LightGBM

每种模型均需要完成基础训练，并使用Optuna进行自动超参数搜索。模型调优以ROC-AUC为核心指标，同时对比Accuracy、Precision、Recall和F1-score。

---

## 2. 数据集说明

模型使用的数据文件包括：

- `train_dataset.parquet`：训练集
- `valid_dataset.parquet`：验证集
- `test_dataset.parquet`：测试集

其中：

- 训练集用于拟合模型参数；
- 验证集用于Optuna参数搜索和模型选择；
- 测试集仅用于最终模型评价，避免测试集信息泄露。

目标变量为：

- `label = 1`：用户未来7天发生购买行为；
- `label = 0`：用户未来7天未发生购买行为。

由于购买样本少于未购买样本，本项目属于类别不平衡二分类问题。

---

## 3. 逻辑回归基础模型

首先构建逻辑回归基础模型，主要参数包括：

- `C = 1.0`
- `penalty = l2`
- `solver = liblinear`
- `class_weight = balanced`
- `max_iter = 2000`

其中，`class_weight="balanced"`用于提高少数类购买样本在训练过程中的权重。

基础模型评价指标保存于：

- `logistic_regression_metrics.csv`

主要评价指标包括：

- Accuracy
- Precision
- Recall
- F1-score
- ROC-AUC

---

## 4. Optuna自动调参

使用Optuna对逻辑回归模型进行自动超参数搜索。

主要搜索参数包括：

- `C`：控制正则化强度；
- `penalty`：选择L1或L2正则化。

搜索范围：

- `C`：0.0001至100，采用对数尺度搜索；
- `penalty`：L1、L2。

Optuna共执行30次参数试验，每次使用一组不同参数训练模型，并计算验证集ROC-AUC。
